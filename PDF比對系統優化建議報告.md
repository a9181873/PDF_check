# PDF 比對系統優化建議報告

## 1. 差異註記色彩
- 所有差異（即使是灰階）皆以單一高對比亮色（如 #FF00FF 或 #FFFF00）標註，避免在灰階或低彩度背景下不明顯。
- 前端/後端渲染遮罩時，統一使用此色碼，並可考慮加上透明度（如 rgba(255,0,255,0.4)）。

## 2. 誤報過濾與比對精度
- 目前有「肉眼看起來一樣但顯示有差異」的情形，常見原因如下：
  - 解析器（如 PyMuPDF、pdfminer、docling）抽取文字方式不同，導致空白、換行、字型、合字（ligature）等細節差異。
  - PDF 內部字型嵌入或 ToUnicode table 差異，造成同字不同碼。
  - 圖像/向量物件壓縮或渲染細節不同。
- 建議：
  - 統一比對時的 PDF 解析引擎（如全用 PyMuPDF 或 pdfplumber），減少跨引擎差異。
  - 文字比對前，先做 Unicode NFKC 正規化、去除多餘空白、合併合字、數字格式統一。
  - 對於純視覺差異（像素級），可用 PyMuPDF 渲染高解析度影像，搭配 OpenCV/SSIM 計算差異遮罩，並用形態學處理去除雜訊。
  - 若文字層比對有差異但視覺層無差異，可進一步檢查字型/字形是否等價，必要時自動降為低置信度或自動過濾。

## 3. 專案設定與比對依據
- 建議將「專案設定」獨立為 JSON 檔，內容包含：
  - 亮色註記色碼（highlight_color）
  - 渲染 DPI（render_dpi）
  - 像素差異門檻（pixel_threshold）
  - 最小差異區塊面積（min_area）
  - 優先解析器（preferred_parser）
  - 比對依據（如「兩檔共通處+核對日期時間點」）
- 比對時可自動記錄兩檔共通欄位（如標題、公司名、日期等）與比對時間，方便日後查驗。

## 4. 快照與比對記錄保存
- 建議每次比對完成後，將下列內容存入專屬目錄（如 snapshots/{task_id}/）：
  - 原始 PDF 檔案
  - 渲染後的每頁快照（PNG/JPG）
  - 差異遮罩圖（亮色標註）
  - 比對 metadata（JSON，含比對參數、共通欄位、時間戳等）
- API 可提供快照與差異圖下載，方便日後稽核或追蹤。

## 5. 其他建議
- 若遇到表格、向量圖、圖片等複雜物件，建議分層比對（文字層、圖像層、表格層），並分別標註差異。
- 可考慮在前端提供「誤報回報」或「人工確認」機制，持續優化自動比對規則。
